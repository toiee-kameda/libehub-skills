# AI Fluency Behaviors Reference

Anthropic の 4D AI Fluency Framework（Dakan & Feller, in collaboration with Anthropic）に基づく、会話分析のための観察チェックリスト。

このリファレンスは `/coach-me` スキルが会話を定性的に分析する際の**観察指標集**として使う。項目ごとにスコアを付けるためではなく、会話のどこにフルーエンシー行動が現れているかを見つけるためのもの。

---

## 使い方（重要）

このチェックリストは **18 項目** あるが、すべてを会話に対してチェックする必要はない。以下の順で使う：

1. **会話のタスクタイプを特定**（SKILL.md 参照）
2. タスクタイプに応じて **重点的に観察する次元・項目** を絞る
3. 観察された行動を、具体的な発言や文脈と紐付けて記録
4. 次元レベルで定性判定（🌱/🌿/🌳）

「観察されなかった」項目は、会話が短い・文脈がないだけの可能性があるため、**安易に低評価としない**。

---

## The 4 Dimensions（正しい 4D）

| 次元 | 英名 | 一言で |
|------|------|-------|
| D1 | **Delegation**（委任） | AIに何を・どこまで任せるかを判断する力 |
| D2 | **Description**（記述） | AIへの伝え方を通じて挙動と出力を導く力 |
| D3 | **Discernment**（識別） | AIの出力・推論を批判的に評価する力 |
| D4 | **Diligence**（誠実） | AI活用に責任を持ち、透明性・倫理・継続学習を実践する力 |

※ 以前の "Direction" は誤り。正しくは **Diligence**。

---

## D1: Delegation（委任）

AIに任せる範囲を戦略的に判断する力。タスクをどう分解するか、どのモード（自動化／思考パートナー／エージェント）で関わるかを意図的に選ぶ次元。

| ID | 項目名 | 観察シグナル |
|----|--------|-------------|
| DG1 | タスクの切り分け判断 | 「ここまでは自分で」「ここからはAIに」という意図的な線引きが見えるか。一括丸投げになっていないか |
| DG2 | AIの能力と限界の理解 | 「最新情報だから検索して」「独自判断が必要だから私が決める」など、AIの得意/不得意を踏まえた依頼か |
| DG3 | 相互作用モードの意図的選択 | 「壁打ちしたい」「自動化してほしい」「継続して代行して」など、モードを選んでいるか |
| DG4 | 実行前のアプローチ相談 | 大きなタスクで、いきなり実行させずに「どう進めるか」を先に議論する行動があるか |

**観察のヒント**：Claude Code や長時間タスクのような**作業代行・自動化型**の会話では DG4 が特に重要。

---

## D2: Description（記述）

目標・期待・条件を明確に言語化してAIの挙動と出力を導く力。AI Fluency Index の研究で**最も頻繁に観察される次元**。

| ID | 項目名 | 観察シグナル |
|----|--------|-------------|
| DS1 | 目的の明確化 | 「〜したい」「〜のために」など、ゴールや成功条件が伝わる表現があるか |
| DS2 | 形式・構造の指定 | 「表で」「箇条書きで」「コードで」「300字で」など、出力形式の指定があるか |
| DS3 | 良い例の提示 | 「〜みたいに」「このサンプルを参考に」など、目指す水準を示す例示があるか |
| DS4 | 対象読者・文脈の共有 | 「初心者向け」「社内向け」「既存の〜に合わせて」など、読み手や場面の情報があるか |
| DS5 | 協働スタイルの設定 | 「間違いがあれば指摘して」「推論を先に説明して」「不確かな部分は明示して」など、対話ルールを先に設定しているか |

**観察のヒント**：**DS5 は AI Fluency Index で「最大の伸びしろ」と指摘された項目**。実行率は全会話の約 30% のみだが、会話全体の質を変える力がある。DS5 が見られた場合は必ず褒めるポイントとして扱う。

---

## D3: Discernment（識別）

AIの出力・挙動・推論を批判的に評価する力。研究が「**アーティファクト作成時にすべての指標が低下する**」と警告した要注意次元。

| ID | 項目名 | 観察シグナル |
|----|--------|-------------|
| DC1 | 反復と精緻化 | 初回回答で終わらず、追質問・修正指示で磨き込んでいるか。複数ターンの継続があるか |
| DC2 | 推論の問い直し | 「なぜ？」「根拠は？」「本当に？」など、結論の理由を問う発言があるか |
| DC3 | 文脈不足の特定 | 「〜については？」「〜の場合はどう？」など、回答に欠けている前提や視点を指摘しているか |
| DC4 | 重要事実の検証 | 「これは正確？」「ソースは？」など、事実やデータのクロスチェックを求めているか |
| DC5 | Polish への警戒 | 見た目が整った出力（動くコード・きれいなレポート・完成っぽいアプリ）に対して、内容の正しさを立ち止まって確認しているか |

**観察のヒント**：
- **DC1 は研究で 85.7% の会話で観察された最頻出行動**、かつ他のすべての指標と最強の相関を示した中核行動。反復が起きていたら必ず評価する。
- **DC5（Polish の罠）** は AI Fluency Index が新しく提示した概念。**アーティファクト生成型の会話では必ず観察**する。見た目の完成度と内容の正しさは別という注意喚起。

---

## D4: Diligence（誠実）

AI活用に責任を持ち、透明性・倫理・継続学習を実践する力。AI Fluency Index が「観察不能だが最も重要」と明言した次元。**会話内で直接観察できる行動は限られる**が、発言の端々に現れる。

| ID | 項目名 | 観察シグナル |
|----|--------|-------------|
| DL1 | 透明性（利用開示） | 成果物にAI使用を明記する意識・発言があるか |
| DL2 | 検証責任 | 「最終確認は自分でする」「このまま出さずに見直す」など、人間が責任を持つ姿勢が見えるか |
| DL3 | 倫理・ポリシー遵守 | 著作権・プライバシー・機密情報・組織ポリシーへの配慮が発言に現れるか |
| DL4 | 継続的学習（Learning Curve） | 「この使い方は次回も使える」「前はこうだった」など、学習の蓄積を意識する発言があるか |

**観察のヒント**：
- D4 の項目は会話ターンが短いと観察できないことが多い。**観察できないこと自体を欠点として扱わない**。
- 逆に少しでも現れていたら強く評価する（普段から意識していないと言葉に出ない次元）。
- **DL2 はアーティファクト生成型の会話で DC5 と並んで重要**。生成物をそのまま使わず「見直す」姿勢があるか確認する。

---

## 次元レベルの定性判定（🌱🌿🌳）

項目スコアは付けず、次元ごとに3段階で判定する：

| 記号 | レベル | 判定基準 |
|------|-------|---------|
| 🌱 | **芽が出ている** | その次元の観察シグナルが1つでも見られた。意識はしている |
| 🌿 | **育っている** | 複数の項目で行動が組み合わさっている。意図的に実践している |
| 🌳 | **根づいている** | 自然に・無意識にできており、工夫の跡が見える。他者に教えられるレベル |

**判定できない場合**：会話が短い・タスクタイプとしてその次元が関与しない場合は、無理に判定せず「—（この会話からは観察しにくい次元）」と表記してよい。

## タスクタイプ別の重点観察次元

SKILL.md で特定したタスクタイプに応じて、以下のように重点を変える：

| タスクタイプ | 重点次元 | 特に見る項目 |
|-------------|---------|-------------|
| 情報収集型 | D2, D3 | DS1（目的）, DC2（推論問い直し）, DC4（事実検証） |
| アーティファクト生成型 | D3, D4 | DC5（Polishの罠）, DL2（検証責任）, DC1（反復） |
| 思考整理・壁打ち型 | D1, D2 | DG3（モード選択）, DS5（協働スタイル） |
| 学習・理解型 | D2, D3 | DS4（文脈共有）, DC1（反復）, DC2（問い直し） |
| 作業代行・自動化型 | D1, D4 | DG4（アプローチ相談）, DG1（切り分け）, DL2（検証責任） |

複数タイプが混在する会話では、最も比重の大きいタイプを主軸に、副次的なタイプの項目も補助的に確認する。

## 研究知見（正確な数値）

出典を明記できるもののみ記載：

1. **DC1（反復と精緻化）は最頻出・最強相関**
   AI Fluency Index (2026) で観察された会話の 85.7% で見られ、他のすべての指標と最強の相関を示した中核行動。
   → コーチング時は反復の有無を最優先で確認する。

2. **DS5（協働スタイル設定）は最大の伸びしろ**
   実行率は全会話の約 30% に留まるが、実行された場合は会話全体の質が顕著に向上する。
   → 見られたら強く評価し、見られなかった場合は次回のヒントとして必ず提示する。

3. **アーティファクト生成時に Discernment の全指標が低下**
   コード・文書などの生成タスクでは D3（DC1〜DC5）の行動頻度がまとめて落ちる傾向。
   → アーティファクト生成型の会話では DC5（Polish の罠）を必ず扱う。

4. **Learning Curve の存在**
   Anthropic Economic Index "Learning Curves" (March 2026) によると、6ヶ月以上の継続利用者は同タスクでも成功率が約 +5pp 高い。
   → 継続的な振り返りが成長に繋がることをフィードバック末尾で自然に伝える。
