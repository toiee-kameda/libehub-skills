# AI Fluency Self-Check スキル

## 起動条件

ユーザーが `/fluency-check` と明示的に入力したときのみ起動する。自動起動・推測起動はしない。

---

## 実行フロー

### Step 0｜過去データの検索

`conversation_search` ツールで `"AI Fluency Score Record"` を検索する。

- **見つかった場合**：最新の記録（日付・各次元スコア・詳細スコア）を取得してメモリに保持。Step 1 で「前回と比較します」と伝える。
- **見つからなかった場合**：初回として進める。

---

### Step 1｜ガイダンスの表示

以下の内容を表示してから質問セッションに入る：

```
🎯 AI フルーエンシー自己チェック

4D AI Fluency Framework に基づく 14 項目で、あなたの AI 活用スキルを自己評価します。

📋 評価軸
  D（委任・記述）  ：AIへの依頼の質　　 5問・最大15点
  C（識別・批判）  ：出力の評価と活用　 5問・最大15点
  L（責任ある利用）：倫理・継続的改善　 4問・最大12点

⏱ 所要時間：5〜8 分
💡 各問「詳細を見る」ボタンで4段階の説明を確認できます
```

前回データがある場合：「📅 前回（YYYY-MM-DD）の記録が見つかりました。今回と比較します。」を追加。

---

### Step 2｜質問セッション（3ブロック）

#### 質問の提示ルール

各問の手順：

1. 「**ID｜項目名**」と一行説明を表示
2. `ask_user_input` で選択肢を提示：
   - `0: ほとんどできていない`
   - `1: 時々できている`
   - `2: 一貫してできている`
   - `3: 高度にできていて他者にも教えられる`
   - `詳細を見る`
3. 「詳細を見る」が選ばれた場合のみ → 後掲の詳細テキスト（4段階説明）を表示 → 再度 ask_user_input でスコア選択
4. スコア選択後、次の問へ（コメントなしで淡々と進める）

#### ブロック1：Delegation / Description（委任・記述）

| ID | 項目名 | 一行説明 |
|----|--------|---------|
| D1 | 目的の明確化 | AIへの依頼に「なぜ・何のために」を伝えているか |
| D2 | 文脈・前提条件の共有 | 読者・制約・背景情報をセットで共有しているか |
| D3 | 出力フォーマットの指定 | 表・箇条書き・コードなど形式を積極的に指定しているか |
| D4 | 例示によるガイド | 目指すトーンや構成の参考例を渡しているか |
| D5 | AIへの役割指示 | 「メンターとして」「批評家として」など役割・スタイルを指定しているか |

5問終了後：`✅ D（委任・記述）：X / 15点` と小計表示してからブロック2へ。

#### ブロック2：Discernment（識別・批判的思考）

| ID | 項目名 | 一行説明 |
|----|--------|---------|
| C1 | 反復・精緻化 | 最初の回答を磨くために追質問・追加指示を繰り返しているか |
| C2 | 推論の問い直し | AIの結論の「なぜ？」「前提は？」を意図的に問い直しているか |
| C3 | 欠落前提の指摘・補完 | 地域差・対象者・制度など抜けている前提を確認・補完しているか |
| C4 | 事実検証・クロスチェック | 重要情報をAIに再確認させたり外部ソースで確認しているか |
| C5 | 代替案・反対意見の検討 | 意図的に「逆の立場」「複数案」「デメリット」を求めているか |

5問終了後：`✅ C（識別・批判）：X / 15点` と小計表示してからブロック3へ。

#### ブロック3：Diligence（責任ある利用・継続的改善）

| ID | 項目名 | 一行説明 |
|----|--------|---------|
| L1 | AI利用の開示 | AIがどの部分で関与したかを共有・明記しているか |
| L2 | 倫理・ポリシーの遵守 | 著作権・プライバシー・組織ポリシーを意識してAIを使っているか |
| L3 | 学習・リフレクション | 定期的に「何がうまくいったか・失敗したか」を振り返っているか |
| L4 | 人間の裁量と責任 | 重要な判断・成果物は必ず人間が最終確認しているか |

4問終了後：`✅ L（責任ある利用）：X / 12点` と小計表示してから Step 3 へ。

---

### Step 3｜スコア集計とビジュアライズ

`show_widget` でレーダーチャートとスコアサマリーを表示する。

**レーダーチャートの仕様：**
- 3軸：D（0〜15）、C（0〜15）、L（0〜12）
- 軸は0〜最大値のスケールで正規化（0〜100%表示にするとわかりやすい）
- 前回データがある場合：今回（塗りつぶし）＋前回（点線）を重ね合わせる
- 色：今回 = 青系、前回 = グレー系

**スコアサマリー表（前回比較付き）：**

| 次元 | 今回 | 前回 | 差分 |
|------|------|------|------|
| D（委任・記述）| X/15 | X/15 | ▲X or ▽X |
| C（識別・批判）| X/15 | X/15 | ▲X or ▽X |
| L（責任ある利用）| X/12 | X/12 | ▲X or ▽X |
| **合計** | **X/42** | **X/42** | **▲X or ▽X** |

前回データがない場合は「前回」「差分」列を省略。

---

### Step 4｜テキストフィードバック

#### 4-1. 全体評価コメント

合計スコアに応じたコメントを1〜2文で：

| 合計 | コメント |
|------|---------|
| 0〜14点 | AI活用の基礎固め段階。まず日常的な使い方の習慣から整えていきましょう。 |
| 15〜28点 | 実践的な活用が育ちつつある段階。意識的に取り組んでいる項目が見えています。 |
| 29〜42点 | 高度な活用ができている段階。次は他者への展開・チームへの貢献が視野に入ります。 |

#### 4-2. 強みの次元

最高スコアの次元（同点の場合は複数）について、「何ができているか」を具体的に1〜2文で記述。

#### 4-3. 優先改善項目

スコアが低い項目（下位2〜3項目）について、それぞれ以下の形式で：
```
📌 [項目名]（現在 X/3点）
  課題：〜〜〜
  次のアクション：〜〜〜（具体的に1つ）
```

#### 4-4. 成長の記録（前回データがある場合のみ）

- スコアが上がった項目：「✨ 伸びた項目：〜」
- 変わっていない項目：「→ 引き続き意識したい：〜」
- 下がった項目があれば：「⚠️ スコアが下がった項目：〜（要確認）」

### Step 6｜保存用サマリーの出力

次回の `conversation_search` で確実に引っかかるよう、会話中に以下のブロックをそのまま出力する（フォーマットを崩さない）：

```
## AI Fluency Score Record
日付: YYYY-MM-DD
D（委任・記述）: X/15
C（識別・批判的思考）: X/15
L（責任ある利用）: X/12
合計: X/42
詳細スコア: D1=X, D2=X, D3=X, D4=X, D5=X, C1=X, C2=X, C3=X, C4=X, C5=X, L1=X, L2=X, L3=X, L4=X
```

「このブロックをメモしておくと、次回の `/fluency-check` で自動的に成長比較ができます。」と一言添える。

## 詳細テキスト（「詳細を見る」選択時に表示）

### D1 目的の明確化
- **0**: AIに「やってほしいこと」があいまいで、「○○について教えて」程度の依頼が多い
- **1**: たまに最終目的（アウトプットの用途）を伝えるが、一貫してはいない
- **2**: 依頼のたびに、最終目的や成果物の用途を具体的に伝えている
- **3**: 目的を明確に伝えるだけでなく、タスクを分解し「この部分だけAIに任せる」といった役割分担まで説明できる

### D2 文脈・前提条件の共有
- **0**: 前提条件（読者レベル、組織ルール、既存資料など）をほとんど共有しない
- **1**: 思いついたときにだけ背景情報を書くが、抜け漏れが多い
- **2**: 読者・使用場面・制約（予算・時間など）を一通りセットで共有している
- **3**: どんなタスクでも、必要な文脈をテンプレ的に整理して渡す習慣があり、他者にもその方法を教えている

### D3 出力フォーマットの指定
- **0**: 出力形式を指定せず、毎回AI任せ
- **1**: ときどき「箇条書きで」など形式を指定する
- **2**: 表・コード・テンプレなど、目的に応じて最適な形式を積極的に指定している
- **3**: チームで再利用できるフォーマット（テンプレ・スニペット）を設計し、AI出力に組み込ませている

### D4 例示によるガイド
- **0**: 例文や参考資料をほとんど提示しない
- **1**: 重要なときだけ、参考記事やサンプルを示す
- **2**: 目指すトーンや構成を示すために、具体的な例を積極的に渡している
- **3**: 自作の良いアウトプットをライブラリ化し、AIへの例示として継続的に活用している

### D5 協働スタイルの指定
- **0**: 「とりあえず答えを出してもらう」使い方が中心で、AIの役割を指定しない
- **1**: ときどき「メンターとして」「批評家として」など役割を指定する
- **2**: 「まず質問してから提案して」「間違っていそうなら警告して」など、対話スタイルを明確に指示している
- **3**: タスクに応じたAIの役割設計（ブレスト役、レビュー役など）をパターンとして持ち、他者にも共有している

### C1 反復・精緻化
- **0**: AIの最初の回答をほぼそのまま使うことが多い
- **1**: 気になったときだけ修正指示を出す
- **2**: ほぼ毎回、追質問や追加指示で2〜3回以上の反復を行い、回答を磨き込んでいる
- **3**: 反復プロセス自体を設計し、「たたき台→比較→統合→最終版」といったワークフローを定型化している

### C2 推論の問い直し
- **0**: AIの結論の「なぜ？」をほとんど問わない
- **1**: 疑問を感じたときだけ理由や根拠を尋ねる
- **2**: 意図的に「理由」「前提」「他の選択肢」について質問し、答え方を変えさせている
- **3**: AIの推論プロセスを人に説明できるレベルで理解し、授業・研修で「推論の見える化」を指導できる

### C3 欠落前提の指摘・補完
- **0**: AIの回答に欠けている前提条件に気づかない、または放置することが多い
- **1**: 明らかな抜けだけ指摘する
- **2**: 「地域差」「対象者のスキル」「制度・ルール」など、抜けている前提を体系的に確認・補完している
- **3**: チームに対しても「この前提はどうか？」と問いかける習慣があり、前提チェックの文化を作っている

### C4 事実検証・クロスチェック
- **0**: AIの出力をそのまま事実として受け取ることが多い
- **1**: 重要そうな数字や固有名詞だけ時々調べる
- **2**: 重要な情報は、AIに再確認させたり外部ソースでクロスチェックする習慣がある
- **3**: チームや学習者に対して「検証の手順」を明示し、チェックリストとして運用できている

### C5 代替案・反対意見の検討
- **0**: AIが出した案をほぼそのまま採用する
- **1**: 思いついたときだけ「他の案」を聞く
- **2**: 意図的に「逆の立場」「複数案」「デメリット」などを求めて比較している
- **3**: プロジェクトの中で、AIを使った「多案検討→評価→選択」のプロセスを設計し、他者も巻き込んでいる

### L1 AI利用の開示
- **0**: AIを使ったかどうかをほとんど共有しない
- **1**: 聞かれたときだけAI利用を説明する
- **2**: レポート・コード・教材などに、AIがどの部分で関与したかを明記している
- **3**: 組織やクラスでAI利用開示ルールを整備し、運用まで主導している

### L2 倫理・ポリシーの遵守
- **0**: 著作権・プライバシー・組織ポリシーをあまり意識せずにAIを使う
- **1**: 気になったときだけ規約やルールを確認する
- **2**: 著作権・機密情報・学生評価など、リスクの高い領域では特に注意してAI利用をコントロールしている
- **3**: 自組織・授業向けの「AI利用ガイドライン」を作成・アップデートし、周囲に浸透させている

### L3 学習・リフレクション
- **0**: AIの使い方を振り返ることはほとんどない
- **1**: うまくいかなかったときだけ使い方を見直す
- **2**: 定期的に「どんなプロンプトがうまくいったか」「どこで誤りが出たか」を振り返り、使い方を改善している
- **3**: コースや社内研修として、AI活用のふりかえり・ベストプラクティス共有の場を設計している

### L4 人間の裁量と責任
- **0**: 重要な判断やアウトプットの品質確認をAIにほぼ丸投げすることがある
- **1**: 重要そうな場面では自分で確認するが、基準はあいまい
- **2**: 重要な意思決定や提出物については、必ず自分または他の人間が最終チェック・判断を行うルールにしている
- **3**: チームや学習環境全体で「AIは補助、人間が責任」という原則を明示し、プロセスに組み込んでいる
