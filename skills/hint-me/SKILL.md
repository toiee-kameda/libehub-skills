# 概要

このスキルは、Claudeのチャットに特化して、特有の機能や、プロンプトの習熟を支援するためのもの。ヒントを求められた時、状況に合わせて柔軟にヒントを提案する。ランダムに3つ選んだり、ユーザーのコンテキストや**4D Frameworkの視点**から判断して、役立ちそうなものを提案する。

## 参照ファイル

まず以下を読み込む：
- `references/hints-reference.md` — 全ヒント一覧（冒頭のインデックスで全IDおよび4D別インデックスを確認できる）

ヒントの選択・提案は、必ずこのファイルを読み込んだ上で行う。

---

## 4D Framework について

このスキルはヒントの選択・提示に **4D Framework** の視点を活用する。

| D | 意味 | シグナル例 |
|---|---|---|
| **Delegation** | どのAIツール・機能・モダリティを選ぶか | 「どれを使えばいい？」「〇〇ができる機能は？」 |
| **Description** | AIへの伝え方・指示の精度 | 「うまく指示できない」「思った通りに出ない」 |
| **Discernment** | 出力の評価・改善判断 | 「これで合ってる？」「どう改善する？」 |
| **Diligence** | 責任・透明性・公開前の検証 | 「公開していい？」「事実確認したい」「AI使用を開示したい」 |

各ヒントには `4d_framework` タグが付いており、ユーザーの状況に合った4Dのヒントを優先して選ぶ。

## 動作フロー

### パターン1：会話がある場合（文脈あり）

1. 直前までの会話内容を分析する
2. ユーザーがやろうとしていること・つまずいていそうな点・改善できそうな点を把握する
3. **4D Frameworkでユーザーの状況を診断する：**
   - ツール・機能の選択に迷っている → **Delegation** 不足
   - 指示がうまく伝わっていない・出力がズレる → **Description** 不足
   - 出力の良し悪しを判断できていない・改善方針が不明 → **Discernment** 不足
   - 公開・共有・事実確認・AI使用開示に関わる場面 → **Diligence** 不足
4. `references/hints-reference.md` を読み込み、診断した4Dに対応するヒントを優先して1〜3個選ぶ
5. 「なぜそのヒントが今の作業に役立つか」と **どの4Dの強化につながるか** を一言添えて提案する

> 多すぎるより少なく。1つに絞れるなら1つで十分。

---

### パターン2：会話の冒頭で単独使用（文脈なし）

1. `references/hints-reference.md` のインデックスを確認する
2. **ランダムに3つ**選ぶ（カテゴリが偏らないよう心がける。4D別に偏らないことも意識する）
3. 「こんなのはどうですか？ご存知ですか？」と軽いトーンで問いかける
4. ユーザーの反応に応じて：
   - 「知っている」「他のを」→ 別の3つをランダムに提案（既出は除く）
   - 「〇〇したい」「〇〇用途で」→ パターン3へ移行
   - 「Delegationを鍛えたい」「Descriptionが弱い」等 → パターン3（4D指定）へ移行

---

### パターン3：用途指定または質問による絞り込み

1. ユーザーの用途・目的が明確でない場合、**1つだけ質問する**：
   - 「どんな場面で使うヒントが欲しいですか？（例：記事を書く、データを分析する、ツールを作る…）」
   - または「鍛えたい4Dはありますか？（Delegation：ツール選択 / Description：指示の精度 / Discernment：出力の評価 / Diligence：公開・検証）」
2. 回答に基づいて `use_case` タグ・`category`・`4d_framework` でフィルタリング
3. 絞り込んだ中から2〜3個を提案する

## ヒントの提示フォーマット

### 文脈ありの場合（パターン1）

```
💡 **[タイトル]**
📊 4D: [該当するD] — [なぜ今の会話・用途に役立つか、1文で]
→ [具体的な使い方・プロンプト例]
```

### 文脈なし・ランダム提案の場合（パターン2）

```
💡 **[タイトル]**
[なぜ役立つか、1文で]
→ [具体的な使い方・プロンプト例]
```

> 文脈なしで4Dラベルを毎回出すと説明が多くなりすぎる。パターン2では省略してよい。

複数提示する場合は番号を振る。⭐マーク（`attention: ⭐`）付きのヒントは「見過ごされがちな機能」なので、積極的に選ぶ価値がある。

---

## 注意事項

- `/hint-me` と明示的に呼び出された場合のみ起動する。自動起動・推測起動は一切しない
- ヒントの押しつけにならないよう、提案は軽いトーンで
- ユーザーが「もう知っている」と言ったヒントは繰り返さない
- 1回の返答で提案するヒントは最大3個まで
- ヒントを提示した後は「他にも見てみますか？」など、継続の意思を一言確認してもよい
- 4D Frameworkの説明を求められない限り、詳細な解説は不要。ラベルを添える程度に留める
